# 项目需求文档

## 概述

CookHero 是一个面向大众的智能饮食助手系统，旨在提供从厨房准备、菜品学习、营养规划、饮食推荐到个性化计划制定的全流程支持。系统整合内置菜谱、用户上传菜谱、食堂菜单以及可能的外卖平台数据，通过大语言模型和检索增强生成(RAG)技术为用户提供烹饪知识问答、美食推荐以及饮食计划制定。

系统支持按需查询和基于用户长期习惯及健康目标的深度个性化推荐，目标用户包括厨房新手、健身或减脂用户、健康饮食关注者、存在过敏或饮食限制的用户、依赖食堂或外部餐饮服务的用户，以及喜欢自制菜品并希望记录管理的人群。

## 功能性需求

### 用户故事

#### 厨房新手用户
- **作为** 厨房新手，**我想要** 快速找到简单菜谱，**以便** 学会基本烹饪技能
- **作为** 新手厨师，**我想要** 了解厨房工具和食材准备，**以便** 安全有序地开始烹饪
- **作为** 学习者，**我想要** 获取烹饪步骤的详细指导，**以便** 避免常见错误

#### 健康饮食用户
- **作为** 健身用户，**我想要** 高蛋白低脂的饮食建议，**以便** 达到减脂增肌目标
- **作为** 健康关注者，**我想要** 营养均衡的餐单推荐，**以便** 维持身体健康
- **作为** 过敏患者，**我想要** 排除特定食材的菜谱，**以便** 安全饮食

#### 食堂/外卖用户
- **作为** 食堂就餐者，**我想要** 了解当日菜单的营养信息，**以便** 做出健康选择
- **作为** 外卖用户，**我想要** 获取餐厅菜品的详细介绍，**以便** 选择适合的食物
- **作为** 预算用户，**我想要** 找到性价比高的饮食方案，**以便** 节约开支

#### 自制菜品用户
- **作为** 自制菜爱好者，**我想要** 记录和分享我的菜谱，**以便** 与他人交流
- **作为** 创意厨师，**我想要** 获取食材搭配建议，**以便** 开发新菜品
- **作为** 家庭主妇/主夫，**我想要** 生成购物清单，**以便** 高效采购

### 业务逻辑

#### 查询处理流程
1. 用户输入查询文本
2. 系统进行意图识别和参数提取
3. 路由到相应处理模块 (RAG/推荐/代理)
4. 执行检索或生成逻辑
5. 返回结构化结果

#### 推荐算法逻辑
1. 收集用户历史行为和偏好
2. 分析菜品特征和营养信息
3. 应用过滤规则 (过敏、饮食限制)
4. 计算相似度和个性化评分
5. 生成推荐列表和解释

#### 计划生成逻辑
1. 获取用户健康目标和约束
2. 计算营养需求配比
3. 选择合适菜品组合
4. 生成时间表和购物清单
5. 支持动态调整和反馈

## 非功能性需求

### 性能需求
- **响应时间**: 普通查询 < 2秒，复杂生成 < 5秒
- **并发用户**: 支持至少 1000+ 并发查询
- **可用性**: 系统可用性 > 99.5%
- **检索精度**: 相关结果召回率 > 85%

### 安全性需求
- **数据加密**: API 通信使用 HTTPS
- **用户隐私**: 个人数据隔离存储
- **访问控制**: 敏感操作需要认证
- **数据脱敏**: 处理用户上传内容时进行隐私保护

### 可扩展性需求
- **模块化设计**: 支持独立部署和升级
- **数据源扩展**: 易于添加新的菜谱和菜单数据源
- **模型升级**: 支持更换 LLM 和 embedding 模型
- **多租户**: 支持企业级部署和定制

### 用户体验需求
- **界面友好**: 简洁直观的操作界面
- **多设备支持**: 桌面、移动端良好适配
- **结果可解释**: 推荐和回答提供依据说明
- **个性化学习**: 系统随使用逐渐了解用户偏好
